Closure example using Click Counter

Global Variable Approach (Without Closure)

Clicks: 0

let counterGlobal = 0;
function updateClickCount() {
  ++counterGlobal;
  document.getElementById('clickCountGlobal').textContent = counterGlobal;
}
Explanation:

The counter is stored in a global variable, meaning other scripts can also modify it. This is not secure for controlling data.

Counter Declared Inside the Function

Clicks: 0

function updateClickCountLocal() {
  let counterLocal = 0;
  ++counterLocal;
  document.getElementById('clickCountInsideFunction').textContent = counterLocal;
}
Explanation:

Here, the counter is reset each time the function is called. This doesn't maintain the count across multiple clicks!

Nested Functions (Without Closure)

Clicks: 0

function countWrapper() {
  let counterNested = 0;
  function updateClickCount() {
    ++counterNested;
    document.getElementById('clickCountNested').textContent = counterNested;
  }
  updateClickCount(); // Only called once, doesn't update on click
}
Explanation:

The inner function has access to the counter, but itâ€™s only invoked once, so the counter won't persist.

Closure Approach (Private Variable)

Clicks: 0

const updateClickCountClosure = (function () {
  let counterClosure = 0;
  
  return function () {
    ++counterClosure;
    document.getElementById('clickCountClosure').textContent = counterClosure;
  };
})();
Explanation:

This is where closure shines! The counter is kept private and can only be modified through the provided function, ensuring data encapsulation.
